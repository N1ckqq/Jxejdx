## 🎮 Руководство пользователя

### Оффлайн-вход

1. Откройте экран **«Учётная запись»**
2. Нажмите **«Добавить аккаунт»**
3. Выберите **«Оффлайн»**
4. Введите имя пользователя (3–16 символов, латиница и `_`)
5. Выберите созданный аккаунт в списке — готово!

### Пакетное скачивание модов

1. Перейдите в **Скачать → Моды**
2. **Долгое нажатие** на любой мод → активируется режим выделения
3. Отмечайте нужные моды галочками
4. Кнопки **«Выбрать все»** / **«Снять выделение»** для удобства
5. Нажмите **плавающую кнопку** (FAB) внизу для скачивания

### Скачивание с зависимостями

1. Откройте страницу мода → выберите версию → нажмите **«Установить»**
2. В диалоге слева вы увидите **зависимости с галочками**:
   - ✅ **Обязательные** — отмечены по умолчанию
   - ⬜ **Опциональные** — отмечайте по желанию
3. Справа выберите **для каких версий** установить
4. Нажмите **«Установить»** — мод и отмеченные зависимости скачаются автоматически
5. Система сама подберёт совместимую версию зависимости (по MC-версии + лоадеру)

### Умная рекомендация RAM

Система анализирует:
- Количество установленных `.jar` файлов в папке модов
- Версию Minecraft (1.12 vs 1.20 — разные потребности)
- Общий объём RAM устройства

И выдаёт рекомендацию:
| Ситуация | Рекомендация |
|----------|-------------|
| Ваниль (без модов) | 512–1024 MB |
| Лёгкие моды (1–20) | 1024–1280 MB |
| Средняя сборка (21–80) | 1536–2560 MB |
| Тяжёлый модпак (80+) | 2560–4096 MB |

### Авто-парсер крэшей

При просмотре крэш-лога система автоматически:
1. **Определяет категорию** — OOM, Mixin-конфликт, отсутствующая зависимость, рендер, native crash
2. **Находит виновный мод** — по пакетам в stacktrace и секции "Suspected Mods"
3. **Предлагает действия** — удалить мод, обновить, увеличить RAM, сменить рендерер

### Бенчмарк рендереров

Тест измеряет базовую производительность OpenGL ES через EGL Pbuffer, затем применяет коэффициенты:

| Рендерер | Коэффициент | Для кого |
|----------|-------------|----------|
| NG-GL4ES | 0.85 | Лучший баланс скорости и совместимости |
| GL4ES | 0.80 | Стабильный, для старых версий |
| Vulkan Zink | 0.70 | Для мощных устройств и MC 1.17+ |
| Freedreno | 0.75 | Оптимизирован для Qualcomm Adreno |
| Panfrost | 0.70 | Оптимизирован для ARM Mali |
| VirGL | 0.45 | Самый совместимый, но медленный |

## 🏗️ Архитектура проекта

```
Jxejdx/
├── ZalithLauncher/          # Основной модуль приложения
│   ├── src/main/java/com/movtery/zalithlauncher/
│   │   ├── game/
│   │   │   ├── launch/              # Запуск игры, GameTimeTracker, SmartRamRecommendation
│   │   │   ├── download/assets/     # Скачивание модов, зависимостей
│   │   │   ├── renderer/            # Рендереры + RendererBenchmark
│   │   │   ├── version/             # Управление версиями, модами, обновлениями
│   │   │   └── addons/              # Мод-лоадеры (Forge, Fabric, Quilt...)
│   │   ├── ui/screens/              # Jetpack Compose UI
│   │   ├── crashlogs/               # CrashReportParser
│   │   ├── setting/                 # Настройки (MMKV)
│   │   └── coroutine/               # Система задач
│   └── src/main/jni/                # Нативный код (EGL bridges, JRE launcher)
├── LWJGL/                   # LWJGL для Android
├── ColorPicker/             # Модуль выбора цвета
├── LayerController/         # Система управления слоями контролов
└── Terracotta/              # P2P мультиплеер (EasyTier VPN)
```

## 🛠 Сборка из исходников

### Требования
- **JDK 17** (Temurin рекомендуется)
- **Android SDK** (API 36)
- **Android NDK 25.2**
- **Gradle 8.13**

### Быстрый старт

```bash
git clone https://github.com/N1ckqq/Jxejdx.git
cd Jxejdx
./gradlew assembleDebug
```

APK будет в `ZalithLauncher/build/outputs/apk/debug/`

### CI/CD

Проект настроен на автоматическую сборку через **GitHub Actions**:
- Триггер: push в `main` или `feature/*`
- Результат: APK как артефакт + автоматический pre-release
